# Buyer Psychology

## Buyer State A: Broken System
- Symptoms:
  - existing stack but poor handoffs
  - attribution distrust
  - lead leakage and execution drag
- Buying intent:
  - wants clarity, control, and reliable throughput
  - values proof and implementation credibility
- Messaging needs:
  - diagnosis of failure points
  - confidence in restructuring and integration
  - clear staged plan, not generic promises

## Buyer State B: Missing System
- Symptoms:
  - ad hoc tools/spreadsheets/manual workflows
  - unclear growth motion and no stable pipeline mechanics
  - overwhelmed by enterprise jargon
- Buying intent:
  - wants first real system, practical setup, and early momentum
  - values scoped path and reduced decision complexity
- Messaging needs:
  - practical foundation path
  - productized/scoped implementation clarity
  - low-friction CTAs that build trust before high-commitment asks

## Shared Psychology Signals
- Trust increases when pages show:
  - "what is broken"
  - "what gets built"
  - "what changes after"
  - "what to do next"

## Drift Risks
- Over-indexing on strategic jargon can alienate missing-system buyers.
- Over-indexing on tactical lists can underserve broken-system restructuring buyers.
