# Page Blueprints

## Purpose
Define preferred narrative and section sequencing for core Darling Martech page types. These are structural defaults, not rigid templates.

## Shared Blueprint Rules
- Start with a high-clarity first screen: problem + promise + action.
- Move from diagnosis -> proof -> system framing -> action.
- Keep sections fewer and stronger; avoid thin filler bands.
- Ensure each major section advances a single narrative job.
- Match CTA intent to trust stage.

## Home Blueprint
1. Hero: problem pressure + strategic positioning + primary CTA.
2. Proof signal band: immediate credibility markers.
3. Problem framing: key bottlenecks with information scent.
4. Diagnostic path: tool-first path for self-identification.
5. Productized system framing: service architecture as systems.
6. Proof depth slice: outcomes tied to methods.
7. Process clarity: what engagement looks like.
8. Final CTA block: diagnostic call or qualifying action.

## Services Hub Blueprint
1. Service system overview: why services are structured as systems.
2. Buyer-path segmentation: Grow / Scale / Build framing.
3. Pillar sections with clear differentiation.
4. Proof linkage inside each pillar.
5. Related tools for diagnostic self-selection.
6. Conversion block with trust-appropriate CTA.

## Work/Proof Hub Blueprint
1. Outcome-first hero with proof framing.
2. Filters by outcome/problem to aid relevance.
3. Curated proof cards with context, not just visuals.
4. "What was broken -> what changed" emphasis.
5. Related services/tools pathways.
6. Qualification CTA for similar problems.

## Project Detail Blueprint
1. Stakes and context.
2. Pre-state: what was broken.
3. Intervention: system built or restructured.
4. Outcomes: metrics and operating impact.
5. Implementation layers/platform references.
6. Related next steps (service/tool/problem).

## Tool Page Blueprint
1. Diagnostic purpose and who it is for.
2. Tool interaction and expected output.
3. Result framing with interpretation guidance.
4. Proof linkage ("this is what outcomes look like").
5. Next-step CTA based on result confidence.

## Process Page Blueprint
1. Process philosophy and accountability framing.
2. Stage-by-stage timeline with deliverable outcomes.
3. Collaboration model and communication cadence.
4. Risk reduction and decision checkpoints.
5. Conversion block for fit conversation.

## Resource Page Blueprint
1. Resource intent and audience.
2. Curated content clusters by buyer need.
3. Practical takeaways over generic thought leadership.
4. Cross-links to tools/proof/services where relevant.
5. Subscription or consultation CTA.
