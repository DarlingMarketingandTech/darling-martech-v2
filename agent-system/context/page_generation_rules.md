# Page Generation Rules

## Mandatory Framing
- Use problem-first and systems-first structure.
- Avoid generic agency filler language.
- State buyer path explicitly:
  - broken-system
  - missing-system
  - or segmented support for both

## Structural Requirements
- Early section: clarify problem and buyer state.
- Middle sections: show system model and proof relevance.
- Closing sections: trust-stage-aligned CTA with clear next step.

## Proof and CTA Expectations
- Every major claim should connect to proof, mechanism, or implementation path.
- CTA must match trust stage and buyer state.
- Include at least one low-friction path for missing-system buyers.

## Practical vs Strategic Language Rules
- Use practical framing for foundation-path pages and system-starved operators.
- Use strategic framing for restructuring/optimization pages.
- If both audiences are present, split pathing clearly instead of blending vague copy.

## IA and Navigation Rules
- Recommendations must include problem -> service -> proof -> tool path logic.
- Flag pages that imply enterprise-only positioning where not justified.
- Flag pages that exclude smaller valid buyers by tone or packaging.

## Quality Gate Signals
- Block if page reads like generic agency brochure.
- Warn if only one buyer path is supported where both are relevant.
- Warn if CTA progression ignores trust ladder.
