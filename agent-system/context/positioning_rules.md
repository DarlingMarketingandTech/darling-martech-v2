# Positioning Rules

## Systems-First Framing
- Frame work as connected systems that improve revenue reliability and execution quality.
- Prefer: problem -> system diagnosis -> implementation path -> measurable outcome.

## Non-Size-Based Qualification
- Do not infer fit from company size.
- Fit is based on:
  - urgency of system problem
  - willingness to operate with accountability
  - commitment to implementation

## Practical vs Enterprise Framing
- Use practical framing when audience likely needs first durable system foundations.
- Use strategic framing when audience already has systems but needs restructuring or integration depth.
- Both can coexist on the same page with clear pathing.

## Anti-Pattern Rules
- Avoid "agency buffet" structure with unprioritized capability lists.
- Avoid vague strategic claims with no operational route.
- Avoid excluding smaller operators by tone, packaging, or CTA architecture.

## Required Copy Behavior
- State what changes operationally after implementation.
- Tie each major claim to proof, mechanism, or explicit next step.
- Preserve operator-accountability voice over team-ambiguity framing.
