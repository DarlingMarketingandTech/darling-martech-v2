# Service Clusters

## Cluster: Build
- **Purpose:** Establish core technical and operational structure.
- **Best for:** Missing-system buyers and system-foundation starts.
- **Outcomes:** Stable website/system layer, foundational stack coherence.
- **Connections:** Problems around weak foundation -> proof showing setup/transformation -> tools that diagnose baseline gaps.

## Cluster: Experience
- **Purpose:** Improve conversion, UX journey flow, and trust progression.
- **Best for:** Both buyer states when conversion friction is high.
- **Outcomes:** Cleaner visitor path, improved decision support, stronger CTA flow.
- **Connections:** Problem pages on conversion/path friction -> proof of conversion lift -> tools exposing bottlenecks.

## Cluster: Brand
- **Purpose:** Clarify market signal and credibility.
- **Best for:** Buyers with unclear positioning or low trust conversion.
- **Outcomes:** Sharper differentiation, stronger trust messaging, coherent narrative.
- **Connections:** Problem pages on weak signal -> proof with trust/visibility outcomes -> resources/tools for messaging diagnostics.

## Cluster: Scale
- **Purpose:** Improve lifecycle operations, CRM architecture, automation, attribution, and throughput.
- **Best for:** Broken-system buyers with fragmented stacks.
- **Outcomes:** Reduced leakage, improved response speed, stronger measurement confidence.
- **Connections:** Problems around disconnected systems -> proof showing operational gains -> tools exposing process/attribution issues.

## Cluster: Grow
- **Purpose:** Improve visibility, demand capture, and qualified pipeline growth.
- **Best for:** Buyers needing reliable demand mechanics.
- **Outcomes:** Better qualified demand and improved conversion to opportunities.
- **Connections:** Problem pages on low visibility/pipeline inconsistency -> proof of demand lift -> diagnostic tools for growth constraints.

## Cluster: Foundation
- **Purpose:** First real system path for smaller/local/system-starved operators.
- **Best for:** Missing-system buyers needing practical setup.
- **Outcomes:** Baseline CRM + funnel + reporting + essential automation.
- **Connections:** Missing-system problems -> practical proof -> low-friction tools/resources and scoped CTA progression.
