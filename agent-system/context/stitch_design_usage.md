# Stitch Design Usage

## Purpose
Define how Stitch-generated visuals are used as system-supporting assets for Darling Martech.

## When to Use Stitch
- Hero backplates and atmosphere visuals.
- Service support imagery that communicates system sophistication.
- Proof backplates and outcome framing visuals.
- Concept composites for tool/system mockups.

## Not for
- Literal UI screenshots pretending to be production product states.
- Generic decorative art disconnected from section messaging.
- Unreviewed assets directly pushed into production.

## Workflow
1. Build an asset brief (`asset-brief-prompt.md`).
2. Generate 2-4 variants in Stitch.
3. Score variants against visual hierarchy and persuasion fit.
4. Export with naming compliance.
5. Map to Cloudinary and integrate safely.

## Style Constraints
- Premium dark material tone.
- Controlled contrast with clear focal area.
- No noisy gradients or unnecessary abstract clutter.
- Composition should leave space for type overlays when needed.
