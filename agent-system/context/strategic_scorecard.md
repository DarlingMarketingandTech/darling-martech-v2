# Strategic Scorecard

## Purpose
This scorecard is used to evaluate pages, page plans, service architecture proposals, and major copy directions.

It exists to prevent the site from drifting into generic agency language, weak buyer-path logic, or surface-level design without commercial clarity.

The scorecard should be used alongside design and page-experience scoring, not as a replacement for them.

## Scoring Scale
Use a 1–10 scale for each category.

- **1–3** = weak / incorrect / generic / misleading
- **4–6** = partially correct but underdeveloped
- **7–8** = strong and commercially useful
- **9–10** = exceptional, differentiated, and tightly aligned to the brand

## Core Categories

### 1. Systems Thinking
Does the page frame the work as connected systems rather than isolated tactics?

Strong signals:
- explains relationships between positioning, site, CRM, follow-up, automation, proof, and attribution
- makes it clear that outcomes come from connected infrastructure
- avoids presenting services as unrelated menu items

Failure modes:
- reads like a list of deliverables
- explains channels without operating logic
- talks about marketing activity without system design

### 2. Revenue Clarity
Does the page connect work to business outcomes instead of vanity metrics?

Strong signals:
- references pipeline, booked consults, signed cases, members, qualified leads, conversion, time saved, or attribution clarity
- frames improvements in commercial terms
- makes the cost of the current problem legible

Failure modes:
- overuses impressions, reach, awareness, or generic “growth” language
- does not explain how the work changes the economics of the business

### 3. Positioning Differentiation
Does the page sound like Darling MarTech specifically instead of a generic agency?

Strong signals:
- one-accountable-operator language
- systems-first framing
- diagnosis before implementation
- operator / infrastructure / accountability themes

Failure modes:
- “full-service agency” language
- generic buzzwords
- no meaningful distinction from common freelancers or agencies

### 4. Buyer Path Coverage
Does the page clearly serve the correct buyer state?

Buyer states:
- broken / fragmented system
- missing / undeveloped system

Strong signals:
- page clearly identifies which state it is serving
- broken-system buyers see complexity and integration competence
- missing-system buyers see practical foundation framing without condescension

Failure modes:
- page speaks only to enterprise complexity
- page accidentally excludes smaller but valid system-starved operators
- page blurs both states without helping either

### 5. Anti-Persona Accuracy
Does the page preserve the corrected anti-persona logic?

Strong signals:
- does not disqualify by size alone
- disqualifies by mindset, refusal of systems, or tactic-shopping behavior
- respects valid small-firm and local-operator system problems

Failure modes:
- implies local or small businesses are invalid by default
- reintroduces outdated “mom and pop = not a fit” logic
- equates budget size with strategic fit without nuance

### 6. Proof Integration
Does the page make proof feel connected and useful?

Strong signals:
- references relevant proof routes or outcomes
- connects the promise of the page to actual documented work
- uses proof as explanation, not decoration

Failure modes:
- big claims without proof path
- proof appears disconnected from the service/problem being discussed
- no evidence trail

### 7. CTA Trust Alignment
Does the page ask for the right level of trust?

Strong signals:
- lower-trust pages offer tools, audits, or education first
- higher-trust pages offer process depth, proof, or direct conversation
- CTA matches the reader’s likely intent and page context

Failure modes:
- stage-1 page only offers “book a call”
- missing-system buyers pushed into enterprise consultation language too early
- no clear next step

### 8. Operator Accountability
Does the page preserve the owner-operator strength of the brand?

Strong signals:
- clear ownership
- direct accountability
- no fake-team inflation
- practical, credible language

Failure modes:
- sounds like a faceless agency
- hides who is actually responsible
- weakens the single-accountable-lead positioning

## Score Bands
### 8.5–10.0
Exceptional strategic alignment.
Proceed unless design or implementation constraints block the work.

### 7.0–8.4
Strong enough to proceed.
May need refinement notes, but no major repositioning required.

### 6.0–6.9
Proceed with caution only.
Strategic weaknesses are visible and should be corrected before broad rollout.

### Below 6.0
Block and rework.
The page or plan is strategically weak, generic, or misaligned.

## Common Strategic Failure Patterns
- generic agency hero copy with no system logic
- service page that lists tactics but never explains the operating model
- page that speaks only to high-complexity buyers and ignores system-foundation buyers
- page that speaks only to simple quick wins and hides deeper capability
- proof page with metrics that are not tied back to problem/service logic
- CTA pattern that jumps straight to consultation without building trust
- anti-persona drift that excludes valid smaller operators

## Use in Validation
A page or page plan should be flagged when:
- systems thinking is below 7
- positioning differentiation is below 7
- buyer path coverage is below 6
- anti-persona accuracy is below 7
- proof integration is below 6

If two or more of those are below threshold, the work should normally be blocked until revised.
