# System Foundation Path

## Purpose
Define how Darling MarTech should speak to and eventually build for businesses that do not need enterprise complexity first, but do need a real operating system.

This file exists so the repo agent, content planning, and future service pages do not treat these buyers as invalid by default.

## Who This Path Is For
- small law firms without internal marketing teams
- local service businesses with fragmented or weak tooling
- smaller healthcare/service operators
- new businesses that need a practical launch system
- owners using template platforms or expensive tools without an integrated workflow

## Core Problem Statement
These businesses do not necessarily have a “marketing channel” problem first.
They often have a **system absence** problem.

Typical reality:
- brochure-style website
- weak or missing CRM usage
- no lead routing logic
- manual follow-up
- missed calls or weak intake
- unclear reviews / reputation workflow
- disconnected booking, email, and reporting

## Positioning Standard
Do not describe this path as “small business marketing help.”

Describe it as:
- system foundation
- first real growth system
- practical infrastructure for capture, follow-up, and conversion
- replacing disconnected tools with a working operating layer

## Offer Shape
The right offer is usually not an open-ended custom build first.

It is a scoped foundation build that can include:
- website or high-conversion landing structure
- CRM setup / cleanup
- forms, intake, and routing
- booking or consultation flow
- review / reputation support
- email or SMS follow-up
- basic attribution and dashboard visibility

## Site Implications
Future site content should support a second clear entry path beyond “complex broken systems.”

Recommended framing examples:
- “We have tools, but nothing works together.”
- “We do not really have a system yet.”
- “Our site exists, but it is not helping the business run better.”

## CTA Implications
This path should use practical, lower-friction CTAs such as:
- see what your system is missing
- map the gaps in your current setup
- build the first real version of your stack
- fix intake, follow-up, and visibility

Avoid jumping too quickly to enterprise or abstract consulting language.

## Proof Implications
When presenting proof for this path, emphasize:
- fewer missed leads
- cleaner booking / intake
- improved conversion
- time saved
- better owner visibility
- simpler stack decisions

## Tooling Implications
Tools for this path should focus on understandable business pain, not technical vanity:
- missed lead calculators
- intake / follow-up health checks
- stack simplification audits
- review / booking flow audits

## Agent Rule
When future content or strategy work discusses anti-personas, do not exclude smaller firms automatically.

The question is:
- does the business have a real system problem?

If yes, the path is valid.
