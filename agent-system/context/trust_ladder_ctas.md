# Trust Ladder CTA Rules

## Trust Stages
- `browse`
- `diagnose`
- `learn`
- `evaluate`
- `ready`

## Base CTA Logic
- **browse:** low-friction educational or diagnostic entry.
- **diagnose:** interactive assessment, score, or guided audit.
- **learn:** proof, process explanation, and mechanism depth.
- **evaluate:** scoped consult, roadmap, or targeted planning call.
- **ready:** implementation engagement and commitment CTA.

## Broken-System Buyer CTA Pattern
- Prioritize diagnosis + restructuring confidence.
- Early CTA examples:
  - "Audit the bottleneck"
  - "See proof of system rebuilds"
- Mid-stage:
  - "Review technical roadmap options"
- High-stage:
  - "Book strategy/implementation call"

## Missing-System Buyer CTA Pattern
- Prioritize practical foundation and low-jargon clarity.
- Early CTA examples:
  - "Start with a system foundation diagnostic"
  - "Run a readiness check"
- Mid-stage:
  - "See how a first system gets built"
- High-stage:
  - "Book a scoped foundation setup consult"

## CTA Anti-Patterns
- Asking for high commitment at browse stage.
- One-size CTA language across both buyer states.
- CTA copy that implies size-gate disqualification.
