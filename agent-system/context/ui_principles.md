# UI Principles: Darling Martech Design Constitution

## Purpose
This document defines the visual and experiential standard for Darling Martech. It is the quality bar for design-aware agent decisions. "Compiles and passes lint" is not enough; output must meet this system.

## Core Positioning
- Premium dark-system visual language, engineered and deliberate.
- Strong editorial hierarchy with clear narrative control.
- Less "agency template"; more productized strategic system.
- Proof-first storytelling over generic claims.
- Visuals support persuasion, not decoration.

## Information Architecture Separation
The system must preserve clear distinctions between:
- Services (what is sold as productized systems)
- Proof/Work (measurable outcomes and credibility)
- Tools (diagnostic utilities)
- Process (how work is executed)
- Resources (education and nurture)

Never blur these sections with generic repeated layouts that reduce information scent.

## Visual Language Rules
- Dark surfaces with restraint and contrast hierarchy.
- Fewer, stronger blocks over many weak cards.
- Stable spacing rhythm (section, group, card, element).
- Deliberate section transitions to signal narrative progression.
- Typography carries authority before decorative assets do.

## Storytelling Rules
- Lead with evidence, outcomes, or diagnosis.
- Claims require proof cues nearby (metric, artifact, named result).
- Each section should answer: "Why does this matter now?"
- Service framing should read as systems, not a menu of tasks.

## CTA Hierarchy Rules
- One primary CTA per section; secondary actions are supportive.
- CTA copy should reflect trust stage (browse, evaluate, qualify, commit).
- Avoid equal visual weight across multiple CTAs in the same viewport.

## Interaction Quality Rules
- Micro-interactions should improve orientation or confidence.
- Motion is subordinate to comprehension and must respect reduced motion.
- Hover/focus states must make clickability and hierarchy obvious.
- Mobile interactions must preserve clarity without hidden critical content.

## Anti-Patterns
- Generic gradient-heavy SaaS treatment without narrative structure.
- Card spam where each card has weak differentiation.
- Decorative visuals that do not support a decision.
- Inconsistent heading scales or arbitrary spacing jumps.
- Flat sections with no pacing, resulting in "wall of content" fatigue.

## Decision Check
Before any implementation, ask:
1. Does this increase clarity or trust?
2. Does this improve hierarchy or only add visual noise?
3. Does this strengthen proof and buyer-path progression?
4. Does this feel productized and strategic rather than templated?
