# Visual Hierarchy Rules

## Priority Stack
1. Message (headline + value framing)
2. Proof (metric/evidence)
3. Action (primary CTA)
4. Supportive context (details, metadata, links)

## Typography
- One clear heading path per section (`h2` -> `h3` -> body).
- Avoid equal-size sibling headings that flatten hierarchy.
- Meta labels and helper copy must be clearly subordinate.

## Layout and Grouping
- Group related elements tightly; separate unrelated groups clearly.
- Use whitespace as structure, not filler.
- Limit simultaneous "high-emphasis" elements per viewport.

## Proof-First Rule
- If a claim is bold, adjacent proof must be visible quickly.
- Metrics should be anchored to context ("what changed" + "for whom").

## CTA Rule
- Every section has at most one primary action.
- CTA appears where decision confidence is highest, not arbitrarily at bottom.

## Anti-Patterns
- Multiple high-contrast blocks competing in same region.
- Repeating identical card shells for dissimilar content types.
- Equal visual treatment for all items regardless strategic priority.
