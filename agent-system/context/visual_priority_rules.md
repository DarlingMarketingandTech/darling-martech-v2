# Visual Priority Rules

These rules apply to design reasoning, asset planning, and design-mode validation.

## Priority Rules
- Prioritize one to two high-value visuals over many decorative visuals.
- Every visual must have a job: clarify, persuade, or reduce friction.
- Hero visuals should clarify the offer or elevate trust, never distract from message hierarchy.
- Service visuals should explain capability, support trust, or improve comprehension.
- Proof visuals should strengthen credibility with concrete context, not generic atmosphere.
- Avoid visual clutter, duplicate motifs, and repetitive decorative card/grid shells.
- Maintain restraint consistent with premium brand goals: deliberate, sparse, high-signal surfaces.

## Decision Heuristics
- If removing a visual does not reduce understanding or trust, it is likely decorative.
- If multiple visuals communicate the same point, keep the strongest and remove the rest.
- Favor sequencing where proof/support visuals appear before or adjacent to key CTAs.
- Reject visuals that increase scan burden without improving decision quality.
