# Work/Proof Model

## Model Purpose
Ensure implementation work preserves the site's proof-driven operating logic across services, tools, and route-level UX.

## Relationship Constraints
- Problem -> Tool -> Proof -> Service should remain coherent.
- Proof pages should maintain measurable outcomes and explicit implementation grounding.
- Service pages should maintain links to relevant proof and problem clusters.

## Media Logic Guardrails
- Treat hero/card/support media as role-specific:
  - Hero media: narrative framing.
  - Card media: scannable credibility.
  - Support media: contextual reinforcement.
- Avoid changing all three media roles in narrow-scope tasks.

## Operational Review Prompts
- What changed in buyer trust progression?
- Which proof/supporting routes were intentionally untouched?
- Did any edit weaken route migration consistency (`/lab` vs `/tools`)?
