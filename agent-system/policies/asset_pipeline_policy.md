# Asset Pipeline Policy

## Goal
Prevent visual chaos by enforcing a formal lifecycle from intent to integration.

## Required Lifecycle
1. Design brief creation
2. Asset prompt/spec generation
3. Asset generation (Stitch/Canva/Figma)
4. Review with rubric (hierarchy, clarity, brand fit)
5. Export naming compliance
6. Cloudinary placement mapping
7. Safe repo integration and QA

## Quality Gate
Do not integrate any asset that lacks:
- Defined purpose
- Defined placement
- Defined dimensions
- Style constraints
- Naming compliance
- Cloudinary mapping

## Review Rubric (minimum)
- Supports section objective
- Improves hierarchy
- Preserves readability
- Fits premium dark-system brand posture
- Avoids decorative-only value
