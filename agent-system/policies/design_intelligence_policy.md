# Design Intelligence Policy

## Policy Objective
All design-related agent actions must optimize for clarity, persuasion, and brand distinctiveness within the Darling Martech system.

## Required Context Inputs
Before design decisions, load:
- `context/ui_principles.md`
- `context/component_quality_bar.md`
- `context/page_blueprints.md`
- `context/interaction_rules.md`
- `context/visual_hierarchy_rules.md`
- `context/reference_site_analysis.md`

## Execution Rules
- Diagnose before modifying.
- Prefer structural clarity upgrades before visual polish.
- Keep proof and CTA architecture central to recommendations.
- Reject changes that increase visual noise or template feel.
- Preserve service/proof/tools/process/resources information boundaries.
