# Strategic Evaluation Policy

## Purpose
This policy governs how the agent system should evaluate strategic content and messaging work before recommending or implementing it.

## Required Context
Before generating or evaluating strategic page work, consult:
- `agent-system/context/strategic_standards.md`
- `agent-system/context/positioning_rules.md`
- `agent-system/context/buyer_psychology.md`
- `agent-system/context/system_foundation_path.md`
- `agent-system/context/strategic_scorecard.md`
- `agent-system/context/service_clusters.md`
- `agent-system/context/problem_service_mapping.md`
- `agent-system/context/trust_ladder_ctas.md`
- `agent-system/context/page_generation_rules.md`

## Mandatory Checks
Any page plan, strategic recommendation, or messaging proposal should be evaluated for:
- systems-first framing
- buyer-state clarity
- anti-persona correctness
- cluster coherence
- proof linkage
- trust-stage CTA fit
- non-generic differentiation

## Hard Blocks
Block or return for rework if the content:
- reintroduces size-based disqualification logic
- reads like generic agency copy
- treats tactics as disconnected services without system logic
- ignores missing-system buyers where they are clearly relevant
- has no proof path or no viable CTA path

## Caution Conditions
Proceed with caution if:
- one buyer state is served clearly but the other is absent where it may matter
- proof is implied but not mapped
- CTA logic is partially correct but weak
- copy is directionally right but still too broad or soft

## Approval Standard
The content should feel:
- operator-led
- commercially clear
- problem-first
- system-minded
- accountable
- usable for real page-building decisions
