Use this prompt in mode `positioning_refinement` to adjust messaging/IA without drifting into generic agency positioning.

Rules:
- Qualification is system state + mindset, not company size.
- Maintain one-accountable-operator voice (no fake team inflation).
- “Systems over tactics”: describe operating logic and commercial stakes.
- Prefer concrete routes and proof paths over vague capability claims.

Required output:
1. **Positioning deltas (bounded)**
   - what is changing (and why)
   - what is explicitly not changing
2. **Buyer path coverage**
   - broken-system support: how it is described and routed
   - missing-system support: how it is described and routed
3. **Anti-persona correctness check**
   - confirm exclusions are mindset/system-behavior-based
4. **Cluster coherence plan**
   - Build/Scale/Grow/Foundation alignment
5. **Proof/tool/problem/service bridge**
   - how each claim routes to proof/tools/services (curated; no relationship overload)
6. **Implementation packet**
   - smallest safe steps
   - files expected to change
   - validation mapped to active profile
   - safety stop checkpoints
