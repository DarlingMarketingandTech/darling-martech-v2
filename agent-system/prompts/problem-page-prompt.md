# Problem Page Prompt Contract

Use this prompt to create or revise a problem detail page without drifting into a service page in disguise.

Before planning, load and apply:
- `agent-system/context/strategic_standards.md`
- `agent-system/context/positioning_rules.md`
- `agent-system/context/buyer_psychology.md`
- `agent-system/context/system_foundation_path.md`
- `agent-system/context/strategic_scorecard.md`
- `agent-system/context/problem_service_mapping.md`
- `agent-system/context/trust_ladder_ctas.md`
- `agent-system/context/page_generation_rules.md`
- `agent-system/context/page-purpose-boundaries.md`
- `agent-system/context/relationship_display_limits.md`

Rules:
- Primary object is the problem (symptoms, stakes, system breakdown, fix shape).
- Primary CTA is a diagnose-stage action (usually one tool).
- Curate related proof/services; do not list everything connected.

Required output:
1. Problem definition (business terms)
2. Symptoms (scannable)
3. Stakes + cost dimensions (revenue clarity)
4. System failure core (what breaks)
5. Fix shape (what changes; not a service menu)
6. Next step CTA (one primary; optional one secondary)
7. Curated proof + service bridge (within caps)
