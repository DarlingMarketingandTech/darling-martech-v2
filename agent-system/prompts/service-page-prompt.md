Use this prompt to create or revise service pages without becoming an encyclopedic capability list.

Rules:
- Primary object is the service: what changes, who it’s for, how it works, outcomes.
- Primary CTA is evaluate-stage (fit/roadmap conversation).
- Do not disqualify by size; disqualify by mindset/system behavior.
- Curate related items (caps apply); route to hubs instead of enumerating.

Required output:
1. Who it’s for (buyer state + anti-persona)
2. What changes (system-level)
3. How the engagement works (sequence + artifacts)
4. Outcomes (commercial, observable)
5. Proof reinforcement (curated; max 3)
6. Next step CTA (one primary + optional secondary)
