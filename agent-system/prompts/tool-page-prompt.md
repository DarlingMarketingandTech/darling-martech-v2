# Tool Page Prompt Contract

Use this prompt to create or revise a diagnostic tool page so it stays tool-first and routes users correctly.

Before planning, load and apply:
- `agent-system/context/strategic_standards.md`
- `agent-system/context/positioning_rules.md`
- `agent-system/context/buyer_psychology.md`
- `agent-system/context/system_foundation_path.md`
- `agent-system/context/strategic_scorecard.md`
- `agent-system/context/problem_service_mapping.md`
- `agent-system/context/trust_ladder_ctas.md`
- `agent-system/context/page_generation_rules.md`
- `agent-system/context/page-purpose-boundaries.md`
- `agent-system/context/relationship_display_limits.md`

Rules:
- Primary object is the tool (what it measures, how to run it, how to interpret results).
- Primary CTA is “run/complete the tool”.
- Secondary CTAs route to proof/service only after results intent is clear.
- Keep copy practical; avoid essay-style content drift.

Required output:
1. What the tool diagnoses (and who it’s for)
2. Inputs + how it works (brief, credible)
3. Output interpretation (what results mean)
4. Routing logic (problem/proof/service next steps; curated caps)
5. CTA plan (one primary + optional one secondary)
