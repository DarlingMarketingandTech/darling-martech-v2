# Brand System — Darling MarTech

## Positioning posture

Darling MarTech is not positioned as a generic agency.

It is:
- systems-first
- problem-led
- proof-backed
- solo-operator led
- technical and commercially grounded

The site should feel:
- dark
- premium
- structured
- technical
- restrained
- authored

Avoid drifting into:
- generic SaaS polish
- loud gradient clutter
- decorative motion for its own sake
- vague agency language

---

## Voice

Use:
- direct language
- precise language
- proof-backed statements
- short, readable sentences

When speaking about service delivery, prefer:
- `I`
- `Jacob`

Avoid:
- `we` for service delivery
- inflated consulting language
- filler phrases

---

## Color direction

Primary palette in active repo use:
- near-black backgrounds
- dark elevated surfaces
- orange for primary accent and action
- teal for secondary/system signal
- proof green for validated metrics and positive proof signals
- off-white for body text

Proof green should stay constrained to:
- proof metrics
- positive validated outcomes

Do not use it as a decorative accent everywhere.

---

## Typography direction

The active repo uses:
- Syne for display
- Inter for body
- JetBrains Mono for data and signal surfaces

Typography should create strong hierarchy between:
- statement
- explanation
- metadata
- proof signals

---

## Visual language

Use:
- layered surfaces
- premium dark materials
- subtle borders
- soft glows only where they reinforce system hierarchy
- asymmetry when it improves composition
- technical metadata labels
- restrained motion

Allowed:
- local CSS modules where scoped premium treatment is the cleanest tool
- Tailwind-first styling across the repo

Avoid stale blanket rules such as “never use CSS modules” when current repo reality already uses local modules appropriately for scoped motion and section treatment.

---

## Motion

Motion should feel like system behavior, not decoration.

Prefer:
- fade
- rise
- stagger
- restrained hover emphasis

Respect reduced motion.

Do not introduce heavy motion without clear value.

---

## Signature quality bar

The site should increasingly feel like:
- a product-level experience
- a system explanation engine
- a premium technical operator brand

Not:
- a portfolio grid
- a generic services brochure
- a template with better copy
